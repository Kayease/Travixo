Travixo Documentation

Premium Travel & Tour Booking Template built with Next.js 16, React 19, TypeScript 5, and Tailwind CSS 4.

Thank you for purchasing Travixo! This documentation will guide you through installation, customization, and deployment of your new travel template.

32 Pages Pre-built pages
90+ Components Reusable UI blocks
100% Responsive Mobile-first design
WCAG 2.1 AA Accessible

Requirements

  • Node.js — v18.17 or later (LTS recommended)
  • npm — v9+ (comes with Node.js)
  • Code Editor — VS Code recommended
  • Git — For version control (optional)

Installation

Step 1: Extract the files

Unzip the downloaded package to your desired directory.

Step 2: Install dependencies

cd travixo
npm install

Step 3: Start the development server

npm run dev

Open http://localhost:3000 in your browser to see the template.

Step 4: Build for production

npm run build
npm start

Environment Setup

Copy the example environment file and update with your values:

cp .env.example .env.local
Variable Description Default
NEXT_PUBLIC_SITE_URL Your production domain URL. Used for SEO metadata, sitemap, robots.txt, and canonical URLs. https://travixo.kayease.com
Note: The NEXT_PUBLIC_SITE_URL variable is used across all pages for canonical URLs, Open Graph metadata, and structured data (JSON-LD). Make sure to set it to your actual domain before deploying.

Colors & Branding

Brand colors are defined as CSS custom properties in app/globals.css and registered with Tailwind via the @theme directive:

Name Hex CSS Variable Tailwind Class
Brand Brown #4B3621 --color-brand-brown text-brand-brown / bg-brand-brown
Brand Orange #F97316 --color-brand-orange text-brand-orange / bg-brand-orange
Brand Cream #FDFBF7 --color-brand-cream text-brand-cream / bg-brand-cream

Changing Brand Colors

To customize the color scheme, edit the :root variables in app/globals.css:

:root {
  --color-brand-brown: #4b3621;   /* Primary text color */
  --color-brand-orange: #f97316;  /* Accent / CTA color */
  --color-brand-cream: #fdfbf7;   /* Background color */
}

SEO & Metadata

Every page includes comprehensive SEO metadata:

  • Title & Description — Unique per page via Next.js Metadata
  • Canonical URLs — Prevent duplicate content (uses NEXT_PUBLIC_SITE_URL)
  • Open Graph — Rich social media previews
  • Twitter Cards — Twitter-specific previews
  • JSON-LD — Structured data for Organization, WebSite, HotelRoom, TouristAttraction
  • Sitemap — Auto-generated at /sitemap.xml
  • Robots.txt — Auto-generated at /robots.txt

Customizing Metadata

Each page's metadata is defined in its layout.tsx or page.tsx file. The site URL is centralized in app/lib/siteConfig.ts:

import { siteUrl } from "@/app/lib/siteConfig";

export const metadata: Metadata = {
  title: "Your Page Title | Travixo",
  alternates: {
    canonical: `${siteUrl}/your-page`,
  },
};

Cart & Wishlist

State management uses React Context with localStorage persistence:

  • CartContext (app/context/CartContext.tsx) — Add/remove items, update quantities, calculate totals
  • WishlistContext (app/context/WishlistContext.tsx) — Save/remove favorites
  • ToastContext (app/context/ToastContext.tsx) — Show success/error/info notifications
Note: Cart and wishlist data persists in the browser's localStorage. In a production app, you would connect these to a backend API.

Accessibility

Travixo follows WCAG 2.1 AA guidelines:

  • Skip to content — Hidden link at page top for keyboard navigation
  • Focus-visible styles — Orange outline ring on keyboard focus
  • ARIA labels — All interactive elements have descriptive labels
  • Semantic HTML — Proper heading hierarchy, landmarks, and roles
  • Alt text — All images have descriptive alternative text
  • Reduced motion — Respects prefers-reduced-motion OS setting
  • Color contrast — Text meets WCAG AA contrast ratios

Animations

Custom animations are defined in app/globals.css:

  • Float animation — Gentle floating effect for hero images
  • Explore fade-in — Tab content transitions
  • Explore image-in — Image reveal animations
  • Hover effects — Bottom-to-top fill on buttons and cards

All animations are automatically disabled when the user has prefers-reduced-motion: reduce enabled in their OS settings.

Build & Deploy

Production Build

npm run build

This creates an optimized production build in the .next/ directory.

Start Production Server

npm start

Static Export (Optional)

To generate a static HTML export, add to next.config.ts:

const nextConfig: NextConfig = {
  output: 'export',
};

Deploy to Vercel

  1. Push your code to a Git repository (GitHub, GitLab, or Bitbucket)
  2. Go to vercel.com and import your repository
  3. Set environment variables (NEXT_PUBLIC_SITE_URL)
  4. Click Deploy — Vercel auto-detects Next.js and configures everything
